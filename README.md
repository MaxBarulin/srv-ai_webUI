# srv-ai webUI

Корпоративный веб-интерфейс для локальной LLM (llama.cpp, Qwen) — единое рабочее
место отдела: чат с моделью, заметки, календарь, поиск по базе знаний (LightRAG).
Модель умеет работать с заметками и календарём через tool calling по запросу на
естественном языке. Разработан по [TZ_srv-ai_webui.md](TZ_srv-ai_webui.md).

Проект рассчитан на закрытый контур (air-gap): без CDN, внешних шрифтов и внешних
API в рантайме; минимум зависимостей; обозримый для аудита ИБ код.

## Возможности

- **Аутентификация и роли** — локальные логин/пароль (bcrypt), сессии в БД,
  rate limit, роли `admin`/`user`, админка пользователей.
- **Чат** — стриминг из llama.cpp (SSE), история, режим «Размышления», markdown с
  подсветкой таблиц и кнопкой копирования, остановка генерации.
- **Заметки и календарь** — личные и общие, полный CRUD, тот же API используют
  инструменты LLM.
- **Tool calling** — серверный агентный цикл: модель создаёт/ищет/меняет заметки и
  события от имени пользователя; подтверждение деструктивных действий; аудит.
- **RAG** — переключатель «База знаний», контекст из LightRAG, блок «Источники».
- **Эргономика** — специализации (режимы чата), обратная связь 👍/👎 с выгрузкой
  в JSONL, серверная очередь к модели, таблицы → Excel, примеры запросов, масштаб
  шрифта.
- **Документы** — загрузка txt/md/csv/docx/xlsx/pdf/png/jpg: текст извлекается и
  присоединяется к запросу, изображения и PDF-сканы уходят модели через vision
  (mmproj), защита от вредоносного содержимого.
- **ИБ** — фильтр персональных данных (регулярки + NER ФИО, офлайн), аудит-лог,
  метрики, экранирование HTML, CSP, CSRF-защита, параметризованные запросы.

## Запуск для разработки

```bash
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
cp env.example .env             # Windows: copy env.example .env
python -m app.create_admin
uvicorn app.main:app --reload --port 8080
```

Открыть http://127.0.0.1:8080/login

Для парсинга PDF-сканов нужен `poppler-utils` (`pdftoppm`) — растеризованные
страницы передаются мультимодальной модели (llama.cpp с mmproj), которая
распознаёт их сама.

## Конфигурация

Все параметры — в `.env` (пример и описание в [env.example](env.example)): порт,
адреса LLM и LightRAG, TTL сессий, подтверждение деструктивных действий, лимиты
загрузки и vision, фильтр ПДн.
