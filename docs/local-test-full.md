# Полный локальный тест на Windows (со всем)

Эта инструкция поднимает интерфейс на твоём компьютере **со всеми функциями**:
чат, заметки, календарь, инструменты, **база знаний (LightRAG)**, **фильтр
персональных данных (Наташа)** и **распознавание PDF как картинок (pypdfium2 + vision)**.

Если нужно просто по-быстрому проверить чат — смотри `local-test-quick.md`.

Делай строго по шагам сверху вниз. Всё, что в рамках `код` — копируй в терминал.

---

## Что должно быть на компьютере

1. **Python 3.11** — https://www.python.org/downloads/
   При установке **галочку «Add python.exe to PATH»**.
2. **Git** — https://git-scm.com/download/win

Проверка (новое окно терминала):
```
python --version
git --version
```

---

## Шаг 1. Забрать проект

PyCharm: **File → Project from Version Control**, вставь URL репозитория → **Clone**.
Открой терминал: **View → Tool Windows → Terminal**. Дальше всё — в нём.

## Шаг 2. Виртуальное окружение
```
python -m venv venv
venv\Scripts\activate
```
Появится `(venv)` в начале строки. Если ошибка «running scripts is disabled»:
```
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```
и снова `venv\Scripts\activate`.

## Шаг 3. Обновить инструменты сборки (важно для Наташи)
```
python -m pip install --upgrade pip setuptools wheel
```

## Шаг 4. Установить ВСЕ зависимости
```
pip install -r requirements.txt
```

**Если появилась ошибка `Failed building wheel for docopt`** (бывает на Windows),
выполни две команды и повтори установку:
```
pip install --use-pep517 docopt
pip install -r requirements.txt
```
Со второго раза дойдёт до конца.

## Шаг 5. Растеризация PDF — уже работает (poppler не нужен)

Режим «PDF как картинку» (страницы → изображения для vision-модели) обеспечивает
пакет **`pypdfium2`** из `requirements.txt` — движок PDFium лежит прямо в wheel,
поэтому на Windows ничего дополнительно ставить и прописывать в PATH не надо.

Резервный путь на `pdftoppm` (poppler-windows) остаётся в коде на случай, если
`pypdfium2` не установлен, но при обычной установке зависимостей он не требуется.

## Шаг 6. Настроить `.env`
```
copy env.example .env
```
Открой `.env` в PyCharm и выставь:
```
LLM_BASE_URL=http://IP_СЕРВЕРА:8000/v1
LLM_MODEL=имя_твоей_модели
RAG_ENABLED=true
RAG_BASE_URL=http://127.0.0.1:7860
PII_FILTER=true
PII_WHITELIST_FILE=./pii_whitelist.txt
```
- `IP_СЕРВЕРА` — адрес сервера с llama.cpp (порт **8000**).
- `RAG_BASE_URL` — адрес твоего локального LightRAG (подставь порт, на котором он
  реально слушает).
- Порт эмбеддингов **8001 указывать не нужно** — к нему обращается сам LightRAG,
  а не наш интерфейс.
- `pii_whitelist.txt` уже есть в проекте (пустой шаблон; позже впишешь туда
  фамилии подписантов нормативки, которые нельзя маскировать).

Сохрани (Ctrl+S).

## Шаг 7. Создать администратора
```
python -m app.create_admin
```
Логин, имя, пароль (**минимум 10 символов**).

## Шаг 8. Запустить
```
uvicorn app.main:app --reload --port 8080
```
Дождись `Application startup complete.` Окно не закрывай.

## Шаг 9. Открыть в браузере
```
http://127.0.0.1:8080/login
```

## Что проверить (полный сценарий)

1. **Чат** — обычный вопрос модели (нужен доступ к серверу :8000).
2. **Фильтр ПДн** — напиши: *«Мой телефон +7 900 123-45-67, подписал Иванов Иван
   Иванович»*. Над сообщением появится плашка **«🛡 Заменено элементов ПДн: N»**.
   > Первый раз модель Наташи подгрузится автоматически (нужен интернет, пара
   > секунд). Дальше работает из кеша.
3. **Инструменты** — *«создай на завтра на 10:00 совещание по нормированию и
   заметку с повесткой»* → появятся событие и заметка.
4. **Документы** — прикрепи `.docx`, `.xlsx`, картинку и PDF-скан (кнопка «📎 Файл»).
5. **База знаний** — включи переключатель «База знаний» в шапке чата (нужен
   запущенный LightRAG), задай вопрос → под ответом будет блок «Источники».
6. **Администрирование** (виден только админу) — «Метрики» (там среди прочего
   счётчики замаскированных ПДн по типам) и «Журнал аудита».

## Управление
- Остановить сервер: `Ctrl+C` в его окне.
- Выйти из окружения: `deactivate`.
- Запустить снова позже:
  ```
  venv\Scripts\activate
  uvicorn app.main:app --reload --port 8080
  ```

---

## Если что-то не так
- **`Failed building wheel for docopt`** — см. шаг 4 (команда с `--use-pep517`).
- **Плашка ПДн не появляется** — проверь `PII_FILTER=true` в `.env` и что сервер
  перезапущен после изменения `.env`.
- **PDF «как картинку» не обрабатывается** — не установлены зависимости: выполни
  `pip install -r requirements.txt` (растеризацию делает `pypdfium2`).
- **«Источники» пустые / ошибка базы знаний** — LightRAG не запущен или неверный
  `RAG_BASE_URL`. Обычный чат при этом продолжает работать.
- **Чат отвечает ошибкой про LLM** — недоступен сервер модели: проверь
  `LLM_BASE_URL` и сеть до сервера (порт 8000).
