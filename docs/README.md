# Инструкции

Три пошаговые инструкции — выбери нужную:

| Инструкция | Для кого | Когда |
|------------|----------|-------|
| [Быстрый локальный тест](local-test-quick.md) | разработчик на Windows | быстро пощупать чат/заметки/календарь у себя на компьютере, без фильтра ПДн и PDF-сканов |
| [Полный локальный тест](local-test-full.md) | разработчик на Windows | проверить **все** функции локально: база знаний, фильтр ПДн (Наташа), PDF как картинки (pypdfium2) |
| [Развёртывание на сервер (с sudo)](../deploy/README-deploy.md) | системный администратор | боевая установка: systemd, firewall, бэкап, офлайн-контур |
| [Развёртывание своими силами (без sudo)](deploy-no-sudo.md) | ты сам, есть SSH-доступ, но нет root | поднять на сервере под своим пользователем через systemctl --user, безопасность через привязку модели к localhost |
| [Быстрый тест на сервере без LightRAG](deploy-test.md) | ты сам, LightRAG ещё нет | временно погасить эмбеддинги и проверить UI на порту 8001, RAG выключен |
| [Стабильный запуск + подключение LightRAG](deploy-stable.md) | ты сам, после успешного теста | сделать сервис, переживающий ребут (systemctl --user + linger); потом подключить LightRAG |

Порядок на практике: сначала «быстрый» или «полный» локальный тест на своём
компьютере, затем — развёртывание на сервер (с sudo — силами сисадмина, либо без
sudo — своими силами).
