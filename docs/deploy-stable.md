# Стабильный запуск (переживает ребут) + подключение LightRAG

Инструкция «для тупых», по шагам. Две части:
- **A** — сделать текущий тестовый запуск (UI на порту 8001, без LightRAG)
  стабильным: работает как сервис, сам поднимается после перезагрузки сервера.
- **B** — что сделать потом, когда будешь подключать LightRAG.

Всё — под своим пользователем, без root/sudo.

---

# ЧАСТЬ A. Сделать стабильно (автозапуск после ребута)

Сейчас UI запущен через `nohup` — он **не переживёт перезагрузку сервера**.
Переведём его на пользовательский systemd-сервис, который стартует сам.

## Шаг 1. Остановить текущий nohup-запуск
```
pkill -f "python -m app"
```

## Шаг 2. Убрать автозапуск эмбеддингов (чтобы не дрались за порт 8001)
UI сейчас живёт на 8001. Эмбеддинги тоже хотят 8001 — пока LightRAG нет, эмбеддинги
не нужны, выключаем их совсем (и из автозапуска тоже):
```
systemctl --user disable --now llama-embed
```

## Шаг 3. Проверить .env
```
nano ~/srv-ai-ui/.env
```
Должно быть:
```
APP_HOST=0.0.0.0
APP_PORT=8001
LLM_BASE_URL=http://127.0.0.1:8000/v1
RAG_ENABLED=false
```

## Шаг 4. Поставить сервис
```
mkdir -p ~/.config/systemd/user
cp ~/srv-ai-ui/deploy/srv-ai-ui.user.service ~/.config/systemd/user/srv-ai-ui.service
systemctl --user daemon-reload
systemctl --user enable --now srv-ai-ui
systemctl --user status srv-ai-ui        # ждём строку: Active: active (running)
```
Проверка:
```
curl -s http://127.0.0.1:8001/api/health   # {"status":"ok"}
```

## Шаг 5. ГЛАВНОЕ для переживания ребута — «linger»

Без этого пользовательские сервисы стартуют только когда ты залогинишься по SSH.
С «linger» они поднимаются сразу при загрузке сервера.

Проверить, включён ли:
```
loginctl show-user "$USER" | grep Linger
```
- `Linger=yes` — **всё готово**, ничего больше делать не надо (у тебя llama так и
  переживает ребут).
- `Linger=no` — попробуй включить:
  ```
  loginctl enable-linger "$USER"
  ```
  Если команда прошла молча — отлично. Если попросила пароль/права — значит без
  админа не получится: тогда сервис поднимется при первом заходе по SSH после
  ребута (либо разово попроси админа выполнить `loginctl enable-linger твой_логин`).

## Готово. Команды на каждый день
```
# посмотреть логи
journalctl --user -u srv-ai-ui -f

# перезапустить (например, после git pull)
systemctl --user restart srv-ai-ui

# обновить UI до свежей версии
cd ~/srv-ai-ui && git pull && systemctl --user restart srv-ai-ui

# остановить / запустить
systemctl --user stop srv-ai-ui
systemctl --user start srv-ai-ui
```
Больше `nohup` и `python -m app` руками запускать не нужно — этим теперь рулит
systemd.

# ЧАСТЬ B. Когда будешь подключать LightRAG

Загвоздка: LightRAG-у нужны эмбеддинги, а они сейчас выключены и их порт 8001
занят твоим UI. Снаружи у тебя открыты только 8000 (llama) и 8001. Решение:
**эмбеддингам внешний доступ не нужен** — их дёргает только LightRAG с того же
сервера. Поэтому переносим эмбеддинги на любой локальный порт, а 8001 оставляем
за UI (чтобы коллеги заходили как раньше).

Схема после подключения:
```
UI               → 0.0.0.0:8001   (снаружи, для людей)   ← НЕ меняем
llama.cpp        → 127.0.0.1:8000 (локально)              ← для UI и LightRAG
эмбеддинги       → 127.0.0.1:8002 (локально)              ← только для LightRAG
LightRAG         → 127.0.0.1:9621 (локально)              ← для UI
```

## Шаг 1. Перевести эмбеддинги на локальный порт (например 8002)
```
nano ~/.config/systemd/user/llama-embed.service
```
В строке запуска поменять хост/порт на `--host 127.0.0.1 --port 8002`
(было `--port 8001`). Затем:
```
systemctl --user daemon-reload
systemctl --user enable --now llama-embed
```

## Шаг 2. Развернуть LightRAG
Разворачиваешь LightRAG (это отдельный сервис, не из нашего репозитория) так,
чтобы:
- он слушал, например, `127.0.0.1:9621`;
- его настройки эмбеддингов указывали на `http://127.0.0.1:8002` (куда ты перенёс
  эмбеддинги в шаге 1).

Проверь, что LightRAG отвечает:
```
curl -s http://127.0.0.1:9621/health   # или другой его эндпоинт статуса
```

## Шаг 3. Включить базу знаний в UI
```
nano ~/srv-ai-ui/.env
```
Поменять/добавить:
```
RAG_ENABLED=true
RAG_BASE_URL=http://127.0.0.1:9621
RAG_MODE=hybrid
# RAG_API_KEY=...   # только если у LightRAG включён ключ
```
Перезапустить UI:
```
systemctl --user restart srv-ai-ui
```
Теперь в чате появится переключатель **«База знаний»**. Включи его и задай вопрос —
под ответом будет блок «Источники».

## Шаг 4 (по ИБ, §13 — на боевом внедрении)
Чтобы к моделям нельзя было ходить в обход UI:
- llama.cpp запускать с `--host 127.0.0.1 --no-webui` (сейчас она на 8000 открыта
  наружу с её web UI — это временно; когда откажетесь от её webUI, закрой порт на
  localhost);
- эмбеддинги уже на `127.0.0.1` (шаг 1) — снаружи недоступны, как и требуется;
- снаружи остаётся открытым только UI (8001) с обязательной аутентификацией.

> Если понадобится, чтобы UI был доступен и на llama-порту вместо её webUI, или
> нужен красивый адрес без `:8001` — это уже правки сети/файрвола, тут может
> потребоваться админ.
