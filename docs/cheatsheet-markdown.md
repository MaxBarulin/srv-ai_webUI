# Шпаргалка по markdown в srv-ai webUI

Что понимает наш рендер — в ответах модели и в предпросмотре заметок.
Рендер свой, минималистичный (по ТЗ §4): весь HTML экранируется, исполняемого
кода в ответах быть не может. Плюс формулы LaTeX через KaTeX (локально).

## Заголовки

```
# Заголовок 1-го уровня
## Заголовок 2-го уровня
### Заголовок 3-го уровня
#### Заголовок 4-го уровня
```

- Поддерживаются `#` … `####` (4 уровня). `#####` и глубже — останутся текстом.
- Внутри ответа они отображаются мельче, чем заголовки интерфейса, — это
  сделано специально, чтобы ответ модели не «перекрикивал» сам UI.
- После `#` обязателен пробел: `#Заголовок` не сработает.

## Начертание текста

| Пишешь | Получаешь |
|---|---|
| `**жирный**` | **жирный** |
| `*курсив*` | *курсив* |
| `` `код` `` | моноширинный `код` |
| `**жирный с *курсивом* внутри**` | работает |

- Только звёздочки: `__жирный__` и `_курсив_` через подчёркивания НЕ работают.
- Зачёркивание `~~так~~`, подсветка `==так==` — НЕ поддерживаются.

## Списки

```
- пункт (также * или +)
- второй пункт

1. нумерованный
2. второй (можно и «2)» со скобкой)
```

- Вложенные списки НЕ поддерживаются — отступы не парсятся, всё сведётся
  в один уровень. Хитрость: имитировать вложенность тире и длинным тире:
  `- пункт — подробность, подробность`.
- Чекбоксы `- [ ]` / `- [x]` не поддерживаются — рисуй `☐` / `☑` символами.

## Таблицы

```
| Колонка А | Колонка Б |
|---|---|
| значение | значение |
| **жирное** | `код` |
```

- Строка-разделитель `|---|---|` обязательна — без неё таблицы не будет.
- Внутри ячеек работает начертание (жирный/курсив/код).
- Широкая таблица не ломает страницу — скроллится внутри своей рамки.
- Таблицы из ответов удобно выделять и вставлять в Excel напрямую.

## Код

````
```python
def hello():
    print("привет")
```
````

- Язык после ``` не обязателен (влияет только на метку блока).
- У блока кода автоматически появляется кнопка «Копировать».
- Внутри блока НИЧЕГО не форматируется — ни markdown, ни формулы. Это
  способ показать разметку «как есть»: оберни в ``` — и `$x$` останется текстом.

## Цитаты и разделители

```
> Цитата — строка, начинающаяся с «> »

---
```

- Цитата одноуровневая; вложенные `> >` не поддерживаются.
- `---` или `***` на отдельной строке — горизонтальная линия.

## Формулы (KaTeX)

| Пишешь | Что это |
|---|---|
| `$T_{баз} = 0{,}048$` | инлайн-формула в строке текста |
| `$$T = \frac{a - b}{c} \times 0{,}006$$` | выключная формула по центру |
| `\( x^2 \)` и `\[ x^2 \]` | альтернативный синтаксис (то же самое) |

Хитрости и грабли:

- **Кириллица в индексах работает**: `$T_{баз}$`, `$V_{расч}$` — норм.
  Для длинного русского текста внутри формулы лучше `\text{...}`:
  `$$0{,}060 \text{ ч/шт.}$$`
- **Десятичная запятая**: пиши `0{,}05` — иначе KaTeX поставит после
  запятой лишний пробел (в математике запятая — разделитель перечисления).
- **Инлайн-доллары с умом**: `$x$` сработает, только если сразу после
  первого `$` и прямо перед вторым нет пробела. «Цена $5 и $10» формулой
  НЕ станет (специальная защита), как и `$ x $` с пробелами.
- Перенос строки внутри инлайн-формулы запрещён; в `$$...$$` — можно.
- Ошибочная формула не ломает ответ — просто останется красным текстом.
- Часто нужное: `\frac{a}{b}` дробь · `x^{2}` степень · `x_{i}` индекс ·
  `\sqrt{x}` корень · `\times` умножение · `\pm` плюс-минус ·
  `\le` `\ge` сравнения · `\rightarrow` стрелка · `\sum_{i=1}^{n}` сумма ·
  `\mathbf{жирное}` в формуле.

## Переносы строк и абзацы

- Пустая строка = новый абзац.
- Одиночный перенос строки внутри абзаца = мягкий перенос `<br>`
  (как в чатах — НЕ как в классическом markdown, где он игнорируется).

## Чего НЕТ (и как жить)

| Нет | Обход |
|---|---|
| Ссылки `[текст](url)` | вставляй URL текстом — его можно скопировать |
| Картинки `![...]` | приложи файл к сообщению — модель видит через vision |
| Вложенные списки | тире внутри пункта: `- пункт — детали` |
| Чекбоксы `- [ ]` | символы `☐ ☑ ✔ ✖` |
| Зачёркивание, подсветка | `➜ УСТАРЕЛО:` / `⚠` пометки текстом |
| HTML-теги | экранируются и видны как текст (защита от XSS — так задумано) |
| `#####`+ заголовки | используй `**жирную строку**` |

## Кнопки рядом с ответом (напоминание)

- **Копировать** — копирует весь ответ в исходном markdown (удобно
  переносить в заметку или Obsidian).
- **Markdown / Рендер** — переключает ответ между красивым видом
  и исходным текстом разметки: подсмотреть, «как это написано».
- **Копировать** на блоке кода — копирует только содержимое блока.
