# Быстрый локальный тест на Windows (без лишнего)

Эта инструкция — чтобы **быстро поднять интерфейс у себя на компьютере** и
пощупать чат, заметки, календарь, загрузку документов. Без фильтра ПДн и без
распознавания PDF-сканов (это в «полной» инструкции — `local-test-full.md`).

Делай строго по шагам, сверху вниз. Всё, что в рамках `код` — копируй и вставляй
в терминал.

---

## Что должно быть на компьютере

1. **Python 3.11** — скачать: https://www.python.org/downloads/
   При установке **поставь галочку «Add python.exe to PATH»** (внизу окна).
2. **Git** — скачать: https://git-scm.com/download/win (жми Next везде).

Проверка (открой новое окно PowerShell или терминал PyCharm):
```
python --version
git --version
```
Должны показаться номера версий. Если «не является внутренней командой» —
Python/Git не в PATH, переустанови с галочкой «Add to PATH».

---

## Шаг 1. Забрать проект

В PyCharm: **File → Project from Version Control**, вставь URL репозитория, нажми
**Clone**. Открой встроенный терминал: **View → Tool Windows → Terminal**.

Дальше все команды — в этом терминале, находясь в папке проекта.

## Шаг 2. Виртуальное окружение
```
python -m venv venv
venv\Scripts\activate
```
После этого в начале строки появится `(venv)`. Если появилась ошибка про
«running scripts is disabled», выполни один раз:
```
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```
и снова `venv\Scripts\activate`.

## Шаг 3. Установить зависимости (облегчённый набор)
```
python -m pip install --upgrade pip
pip install -r requirements-core.txt
```

## Шаг 4. Настроить подключение к серверу
```
copy env.example .env
```
Открой файл `.env` в PyCharm (он в корне проекта) и поправь строки:
```
LLM_BASE_URL=http://IP_СЕРВЕРА:8000/v1
LLM_MODEL=имя_твоей_модели
PII_FILTER=false
```
- Вместо `IP_СЕРВЕРА` — адрес сервера, где запущена llama.cpp (порт **8000**).
- `PII_FILTER=false` — так и оставь (в быстром тесте фильтр не нужен).
- Строку про `RAG_BASE_URL` пока можно не трогать (базу знаний проверим отдельно).

Сохрани файл (Ctrl+S).

## Шаг 5. Создать администратора
```
python -m app.create_admin
```
Введёшь логин, отображаемое имя и пароль (**минимум 10 символов**).

## Шаг 6. Запустить
```
uvicorn app.main:app --reload --port 8080
```
В терминале появится строка `Application startup complete.` — значит, работает.
**Это окно терминала не закрывай**, пока тестируешь.

## Шаг 7. Открыть в браузере
Перейди по адресу:
```
http://127.0.0.1:8080/login
```
Войди под логином/паролем, которые создал на шаге 5.

---

## Что можно проверить
- **Чат** — задай вопрос модели (нужен доступ по сети к серверу и его порту 8000).
- **Заметки / Календарь** — создай пару записей.
- **Инструменты** — напиши в чат: *«создай на завтра на 10:00 совещание по
  нормированию и заметку с повесткой»* — должны появиться событие и заметка.
- **Документы** — прикрепи в чате `.docx`, `.xlsx`, `.txt` или картинку (кнопка
  «📎 Файл»).

## Как остановить
В терминале, где работает сервер, нажми `Ctrl+C`. Чтобы выйти из окружения —
команда `deactivate`.

## Запустить ещё раз позже
```
venv\Scripts\activate
uvicorn app.main:app --reload --port 8080
```

---

## Если что-то не так
- **Браузер пишет «не удаётся подключиться»** — проверь, что в терминале сервер
  запущен и висит на `http://127.0.0.1:8080`.
- **Чат отвечает ошибкой про LLM** — недоступен сервер модели: проверь `IP_СЕРВЕРА`
  в `.env` и что с твоего компьютера есть сеть до него (порт 8000).
- **PDF-сканы (картинки) не грузятся** — это ожидаемо в быстром тесте; для них
  нужен poppler, см. полную инструкцию `local-test-full.md`.
