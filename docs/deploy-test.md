# Быстрый тест на сервере БЕЗ LightRAG (на порту эмбеддингов)

Ситуация: LightRAG ещё не развёрнут, на сервере крутятся только llama.cpp (порт
**8000**, её web UI открыт другим людям) и сервис эмбеддингов (порт **8001**,
доступ только у тебя). Чтобы протестировать свой веб-интерфейс, **временно гасим
эмбеддинги** и занимаем их порт 8001 под UI. База знаний (RAG) на время теста
выключается — эмбеддинги для самого UI не нужны, они нужны только LightRAG.

Когда закончишь тест — вернёшь эмбеддинги обратно (см. конец файла).

---

## 1. Обновить код
```
cd ~/srv-ai-ui
git pull
```

## 2. Освободить порт 8001 (выключить эмбеддинги)
```
systemctl --user stop llama-embed
```
Проверить, что порт свободен (не должно быть строки с :8001):
```
ss -ltn | grep 8001 || echo "порт 8001 свободен"
```

## 3. Настроить `.env` под тест
```
nano ~/srv-ai-ui/.env
```
Выставить эти значения:
```
APP_HOST=0.0.0.0
APP_PORT=8001
DATA_DIR=./data
LLM_BASE_URL=http://127.0.0.1:8000/v1
LLM_MODEL=имя_твоей_модели
LLM_CONTEXT_SIZE=32768        # свой n_ctx (для показа % контекста)
RAG_ENABLED=false             # LightRAG нет — переключатель «База знаний» скрыт
PII_FILTER=false              # для быстрого теста; true — если поставил natasha
```

## 4. Запустить UI

Проще всего — на переднем плане (пока открыт SSH):
```
~/srv-ai-ui/venv/bin/python -m app
```
Увидишь `Uvicorn running on http://0.0.0.0:8001`. Останавливать — `Ctrl+C`.

Чтобы работал и после закрытия SSH:
```
nohup ~/srv-ai-ui/venv/bin/python -m app > ~/srv-ai-ui/app.log 2>&1 &
```
Остановить потом: `pkill -f "python -m app"`

## 5. Проверить
На сервере:
```
curl -s http://127.0.0.1:8001/api/health      # {"status":"ok"}
```
С рабочего компа (браузер):
```
http://192.168.220.154:8001/login
```
Логин/пароль — админ, которого создал через `python -m app.create_admin`.

---

## Вернуть эмбеддинги после теста
```
pkill -f "python -m app"                # остановить UI, освободить 8001
systemctl --user start llama-embed      # вернуть эмбеддинги на 8001
```

## Когда будешь делать «по-полному»
- UI переносишь на отдельный свободный порт (например 8080), эмбеддинги остаются
  на 8001.
- Разворачиваешь LightRAG, в `.env` ставишь `RAG_ENABLED=true` и
  `RAG_BASE_URL=http://<хост-lightrag>:<порт>`.
- Ставишь как постоянный сервис — см. `docs/deploy-no-sudo.md` (раздел про
  `systemctl --user`).
